Icons

One curated set on a 24×24 grid, single 2px stroke, themed with currentColor. There are three ways to render a glyph — all draw from the same source (static/icons/src/), so an icon looks identical no matter which you use. Pick by where the icon lives.

Which technique to use

UseWhenWhy
$$:icon() macro Server-rendered .html templates. The default. Plain inline <svg> against the page-inlined sprite. Paints at first byte — no JavaScript, no layout shift, no flash.
Fragment import
icons.generated.js		 Inside a web component's own (shadow-DOM) template. Shadow roots can't reach the page sprite via <use>, so import the glyph and inline it in your own <svg>. Paints with the component.
<ol-icon> element Icons you insert from client JS, or consumers who only have HTML + the custom element. Self-contained custom element. The box is reserved (no layout shift), but the glyph appears only after the JS upgrades it — so prefer the macro whenever the markup is server-rendered.

Usage

In a Templetor template (preferred). Pass label for an accessible name; without it the icon is aria-hidden:

$$:icon("search")
$$:icon("globe", size="lg")
$$:icon("globe", label=_("Language"))

Inside a Lit component — import the fragment and wrap it in your own <svg>, which controls size and stroke-width via the component's CSS:

import { search } from './icons.generated.js';
html`<svg class="icon" viewBox="0 0 24 24">$${search}</svg>`

As a custom element (client-side / consumers):

<ol-icon name="search"></ol-icon>
<ol-icon name="globe" size="lg" label="Language"></ol-icon>

What you control

Size. Three tokens — sm 16px, md 20px (default), lg 24px — defined as --icon-size-* in rem, so they scale with the user's font size. The macro and <ol-icon> take size="sm|md|lg"; <ol-icon> also accepts a raw pixel number (size="32") for one-offs. When you import a fragment, you set width/height on your own wrapping <svg>.

Color. Every glyph is drawn with currentColor, so an icon is simply the text color of wherever it sits. To recolor one, set color on the icon or any ancestor — e.g. a button with color: var(--red) gets a red icon. No fill or stroke overrides needed.

Stroke weight. A single 2px stroke across the whole set — this is intentionally not a knob. (A fragment import can override stroke-width on its wrapper if a component genuinely needs to, but that's the exception.)
