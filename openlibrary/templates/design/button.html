The ol-button component is a presentation-only button primitive. It renders a real <button> into the light DOM, so styles apply pre-hydration without a flash. It carries no application logic — variants, sizes, and a loading state only.

Form submission

Set type="submit" and place the button inside a <form>. The component renders a real <button type="submit">, so native validation, the form's submit event, and Enter-to-submit from input fields all work without any extra wiring.

<form>
  <input required>
  <ol-button type="submit">Submit</ol-button>
</form>

Events

No custom events. Use the native click event; it bubbles from the inner <button>.

Rendering notes

The component uses light DOM — there is no shadow root. Styles live in a plain stylesheet (components/ol-button.css) that is loaded render-blocking by every page entry point that uses the button. Pre-upgrade the host element is styled via attribute selectors so the button is visible immediately; once upgraded, the component inserts a real <button> child and the host collapses to a transparent wrapper.
