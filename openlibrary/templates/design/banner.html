Banner

The ol-banner component is a callout-style announcement banner. The announcement is server-rendered into the default slot — already translated, visible to search engines — while the component owns the chrome: variant styling, icon, dismissal, and cookie persistence. There is intentionally no entrance animation; banners appear on every page view.

Variants

Use variant for semantic variants. Each pairs a tinted surface with a filled icon circle — the same icon treatment as ol-toast.

<ol-banner>...</ol-banner>
<ol-banner variant="success">...</ol-banner>
<ol-banner variant="warning">...</ol-banner>
<ol-banner variant="danger">...</ol-banner>
Open Library is a non-profit project of the Internet Archive. Your import finished — 42 books added to your reading log. Open Library will be briefly unavailable on June 12 for maintenance. Search is currently degraded. Some results may be missing.

Appearance

Use appearance to control the frame. outlined (default) has a border and rounded corners. plain removes both — useful when the banner abuts the edges of other UI, like the top of the page or a card.

<ol-banner variant="warning">...</ol-banner>
<ol-banner variant="warning" appearance="plain">...</ol-banner>
Outlined (default): border and rounded corners. Plain: no border, no border radius.

Dismissible

Add dismissible for a close button. Without cookie-name, dismissal only lasts for the current page. Content comes from the page template, so links and emphasis work as usual.

<ol-banner dismissible label-close="$$_('Close')">
  <a href="/collections">Explore our new collections</a>
</ol-banner>
Explore our new collections

Custom icon

Slot a custom icon via slot="icon" to replace the variant's built-in glyph.

<ol-banner>
  <svg slot="icon" ...>...</svg>
  Set your Yearly Reading Goal
</ol-banner>
Set your Yearly Reading Goal

Persisted dismissal

The component owns no persistence — it only fires ol-banner-dismiss with its dismiss-id. Two pieces of site plumbing make dismissals stick: the template guards rendering on the dismissal cookie (so dismissed banners are never served), and a site-level listener (js/banner/index.js) POSTs dismissals to /hide_banner, which sets that cookie. Per-banner TTL rides on the element as data-cookie-duration-days — an OL convention, not component API. Dismiss this banner, reload, and it stays hidden until reset.

$$if cookies().get('yrg26') != '1':
  <ol-banner dismiss-id="yrg26" data-cookie-duration-days="365" dismissible>
    <a href="/account/books/already-read/year/2026">Set your 2026 Yearly Reading Goal</a>
  </ol-banner>
$if cookies().get('ol-banner-demo') != '1': Set your 2026 Yearly Reading Goal

Dismiss event

The banner fires ol-banner-dismiss once when dismissed, before it is removed from the DOM. Banners without a dismiss-id are page-only: they still fire the event, but the site listener ignores them.

banner.addEventListener('ol-banner-dismiss', (e) => {
  console.log(e.detail.dismissId);
});
