Toast

The ol-toast component is a transient notification message. It announces itself to screen readers, auto-dismisses after a timeout (pausing on hover/focus), and removes itself from the DOM when closed. Message content is set via the message and description attributes, already translated; rich content can be slotted instead. The component's only owned string is the close button label, overridable via label-close.

Types

Use type for semantic variants. Errors are announced assertively (role="alert").

showToast(i18nStrings.savedToReadingLog);
showToast(i18nStrings.subjectsUpdated, { type: 'success' });
showToast(i18nStrings.listUpdateFailed, { type: 'error' });

Description and rich content

Use description for an optional secondary line. For uncommon rich content (links, custom markup), author the markup in an inert <template> element in the page template — where $$_() extraction works — and pass that element to showToast(); its content is cloned into the toast. Node(s) built in JS also work. Nothing is ever parsed from a runtime string as HTML.

showToast(i18nStrings.relaunchToUpdate, { description: 'v1.11187.4', persistent: true });

<!-- Rich content: write the HTML in the page template... -->
<template id="save-error-toast">
  $$_("Could not save.") <a href="/help">$$_("Get help")</a>
</template>

// ...and pass the template element to showToast()
showToast(document.querySelector('#save-error-toast'), { type: 'error', persistent: true });
Could not save. Get help

Imperative use (fixed bottom-center stack)

JavaScript callers use the exported showToast() helper, which stacks toasts in a shared fixed <ol-toast-region> anchored to the bottom-center of the viewport. New toasts slide up from below; older ones slide up to make room, forming a simple vertical list. Toasts dismiss themselves after timeout milliseconds (default 4000) unless persistent. Hover or focus the list to pause every toast's dismiss timer. The message is set as an attribute — always text, never HTML — and should already be translated.

import { showToast } from './OlToastRegion.js';

showToast(i18nStrings.updateSuccess, { type: 'success' });
showToast(i18nStrings.failedSubmitCheckIn, { type: 'error', persistent: true });

Close event

The toast fires ol-toast-close once when it begins closing, with the reason in event.detail.reason.

toast.addEventListener('ol-toast-close', (e) => {
  console.log(e.detail.reason); // "timeout" | "close-button" | "programmatic"
});
