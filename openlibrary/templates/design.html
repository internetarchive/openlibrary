Pagination

The ol-pagination component provides support for both event-based and URL-based navigation.

Event-based

When a page is clicked, the component fires an update:page event with the page number in event.detail.

<ol-pagination
  total-pages="10"
  current-page="1">
</ol-pagination>

Selected page: 1

URL-based Navigation

When base-url is provided, pagination renders anchor tags for SEO-friendly links.

<ol-pagination
  total-pages="50"
  current-page="5"
  base-url="/search?q=example">
</ol-pagination>

First page (no previous arrow)

Middle page (both arrows visible)

Last page (no next arrow)

3 pages

5 pages (no ellipsis needed)

100 pages with ellipsis:

Arrows mode

When total pages is unknown, use arrows mode to show only previous/next navigation.

<ol-pagination
  mode="arrows"
  current-page="3"
  has-next-page>
</ol-pagination>

Arrows mode (has next page)

Arrows mode (no next page)

Arrows mode (first page, has next)

Chip Group

The ol-chip-group component is a flex-wrap container that provides consistent spacing for groups of chips.

Default (medium gap)

Chips are spaced with an 8px gap by default.

<ol-chip-group>...chips...</ol-chip-group>
Fiction Science History Poetry Drama

Small gap

Use gap="small" for tighter spacing (4px).

<ol-chip-group gap="small">...chips...</ol-chip-group>
Fiction Science History Poetry Drama

Large gap

Use gap="large" for wider spacing (12px).

<ol-chip-group gap="large">...chips...</ol-chip-group>
Fiction Science History Poetry Drama

Wrapping behavior

Chips automatically wrap to the next line when they exceed the container width.

Fiction Science History Poetry Drama Essays Biography

With mixed chip states

Chip groups work with any combination of chip sizes, states, and link chips.

Fiction History Science Poetry

Chip

The ol-chip component is a pill-shaped interactive element for filters, tags, and selections. It fires an ol-chip-select event on click.

Default (medium)

Basic chips at the default medium size.

<ol-chip>
  Fiction
</ol-chip>
Fiction Science History

Selected state

A selected chip displays a checkmark icon and uses an active color scheme.

<ol-chip selected>
  History
</ol-chip>
Fiction History Science

Small size

Use size="small" for a compact chip.

<ol-chip size="small">
  Poetry
</ol-chip>
Poetry Drama Essays

With count

Use count to display a number alongside the label.

<ol-chip count="42">
  Fiction
</ol-chip>
Fiction History

As a link

When href is provided, the chip renders as an anchor tag instead of a button.

<ol-chip
  href="/subjects/fiction"
  count="76">
  Fiction
</ol-chip>
Fiction History

Interactive toggle

Clicking a chip fires an ol-chip-select event. Toggle the selected state by listening to the event.

<ol-chip id="demo-chip">
  Toggle me
</ol-chip>
Toggle me Toggle me too

Domain variants

Use variant to color a chip by the kind of thing it represents (language, subject, genre, author, place, or a neutral facet). The chip maps the variant to a soft-tint palette internally.

<ol-chip variant="language">
  English
</ol-chip>
English Science Fiction Memoir Ursula K. Le Guin San Francisco Read now

Removable filter pills

Combine variant with selected to make a removable, category-colored filter pill. The selected state adds a close icon; listen for ol-chip-select to remove the filter.

<ol-chip variant="language" selected>
  English
</ol-chip>
Read now English French

Markdown Editor

The ol-markdown-editor component is a WYSIWYG editor built on Tiptap that outputs Markdown. It syncs its content to a hidden target textarea. A 'View source' toolbar toggle is always available so editors can drop into a raw markdown textarea when WYSIWYG round-trips are awkward.

Basic usage

Provide a target-id pointing to an existing textarea. The editor reads initial content from the textarea and syncs changes back to it.

<textarea id="my-input">
  **Hello** world
</textarea>
<ol-markdown-editor
  target-id="my-input">
</ol-markdown-editor>

Mixed Markdown and HTML

Add the enable-html-block attribute to expose the HTML block button in the toolbar. HTML blocks appear with a preview and an Edit button to modify the source.

<ol-markdown-editor
  target-id="my-input"
  enable-html-block>
</ol-markdown-editor>

Code blocks and inline code

Add the enable-code attribute to expose the inline code and code block buttons in the toolbar. Code support is off by default because only the wiki page renderer is guaranteed to render fenced code blocks.

<ol-markdown-editor
  target-id="my-input"
  enable-code>
</ol-markdown-editor>

Change event

The editor fires an ol-markdown-editor-change event on every change. The raw Markdown string is available in event.detail.value.

<ol-markdown-editor
  target-id="my-input"
  height="100px">
</ol-markdown-editor>

<script>
  editor.addEventListener('ol-markdown-editor-change', (e) => {
    console.log(e.detail.value);
  });
</script>
